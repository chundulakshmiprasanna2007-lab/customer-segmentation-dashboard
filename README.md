## 🎯 Project Objective

The objective of this project is to transform raw customer spending data stored in a MySQL database into meaningful customer segments and visual analytics — replacing manual classification with a fully automated, real-time system.

Customer categories include:

| Segment | Spending Threshold | Business Action |
|---|---|---|
| 🔴 High Value | ₹5000 and above | Premium membership, VIP offers |
| 🟠 Medium Value | ₹2000 – ₹4999 | Loyalty rewards, referral bonuses |
| 🟢 Low Value | Below ₹2000 | Discount coupons, win-back campaigns |

This helps businesses understand customer behavior, target marketing efficiently, and improve overall revenue.

## 🗄️ Database Structure

### Tables

| Table | Purpose |
|---|---|
| `Customers` | Stores customer details (name, age, gender, income, location) |
| `Transactions` | Stores all purchase transactions linked to customers |
| `Segments` | Defines segment names and descriptions |
| `TransactionLog` | Auto-filled by trigger when transaction > ₹5000 |
| `FirstPurchaseLog` | Auto-filled by trigger on customer's first purchase |

### SQL Components

| Component | Name | Purpose |
|---|---|---|
| View | `CustomerSpending` | Calculates total spending per customer using JOIN |
| View | `AboveAverageSpenders` | Nested query — finds customers above average spending |
| Stored Procedure | `GetCustomerSegment()` | Classifies all customers using CASE logic |
| Trigger | `after_big_transaction` | Logs significant transactions automatically |
| Trigger | `after_first_purchase` | Detects and logs first-time buyers automatically |

## 🚀 Execution Steps

### Step 1 — Database Setup

Open **MySQL Workbench** and execute the SQL file:

```sql
SOURCE database.sql;
```

This will:
- Create the `customer_segmentation` database
- Create all 5 tables with relationships
- Insert sample customer and transaction data
- Create 2 views, 1 stored procedure, and 2 triggers

---

### Step 2 — Compile Java Files

Open terminal in the project folder and run:

```powershell
javac -cp ".;mysql-connector-j-9.6.0.jar" CustomerSegmentationApp.java
javac -cp ".;mysql-connector-j-9.6.0.jar" Server.java
```

---

### Step 3 — Start the Project

**Option A — One Click (Windows):**

Double-click `start.bat` — it automatically starts all servers and opens the dashboard.

**Option B — Manual (3 terminals):**

```powershell
# Terminal 1 — Java backend server
java -cp ".;mysql-connector-j-9.6.0.jar" Server

# Terminal 2 — Python frontend server
python -m http.server 3000

# Terminal 3 — Refresh data (run when needed)
java -cp ".;mysql-connector-j-9.6.0.jar" CustomerSegmentationApp
```

---

### Step 4 — Open Dashboard

Open your browser and go to:

```
http://localhost:3000/index.html
```

## 🔄 How the System Works — End to End

```
User fills Add Customer form in browser
              ↓
JavaScript validates input and sends to Java Server (port 7070)
              ↓
Java inserts into MySQL using PreparedStatement (JDBC)
              ↓
MySQL Triggers fire automatically:
  → after_first_purchase logs new buyer
  → after_big_transaction logs if amount > ₹5000
              ↓
Java calls GetCustomerSegment stored procedure
              ↓
Stored Procedure reads CustomerSpending view
              ↓
CASE logic assigns segment to every customer
              ↓
Java writes real data to customers.js and data.js
              ↓
Browser auto-refreshes after 3 seconds
              ↓
Dashboard shows updated customer with correct segment
```
